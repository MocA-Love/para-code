# Para Code 更新履歴

Para Code が本家 VS Code に加えた変更の一覧です。新しいバージョンが上に来ます。

## paracode-41（2026-07-13）

### 改善

- コマンドプリセットのボタンは、押すたびに常に新しく実行されるようになりました（従来は実行中だと再実行できず、実行中のターミナルへフォーカスが移動していました）。あわせてボタンやドロップダウンの「●（実行中）」表示を廃止し、シンプルな実行ボタンになりました
- コマンドプリセットの「名前を表示」オプションが、アイコンの代わりではなくアイコンに加えて名前を表示するようになりました
- ターミナルの入力候補リストで、→キーを押すと選択中の候補を確定できるようになりました。候補リストが出ていないときの→は従来どおりシェル側の薄い文字（オートサジェスト）の確定として機能します
- ターミナルの入力候補に、いま使っているセッションで実行したばかりのコマンドも反映されやすくなりました（シェルの履歴ファイルを定期的に読み直すようにしました）。日本語などを含むコマンド履歴が文字化けせずに候補に出るようになりました
- ターミナルの候補リストにシェルの薄い文字（オートサジェスト）を重複して表示しないようにしました。薄い文字はシェル、候補リストはエディタ、と役割が分かれて見やすくなります
- モバイルアプリのホーム画面でエージェントの行を長押しすると、背景が暗くぼかされ、対象の行だけが浮き上がって表示されるようになりました（iOSのコンテキストメニューと同じ作法）。どの行に対するメニューかが一目で分かります
- モバイルアプリのエージェント詳細画面で入力中のテキストが、別のタブや画面へ移動して戻ってきても消えずに残るようになりました。下書きはエージェントごとに分けて保持されるため、別のエージェントの入力欄に混ざることはありません（送信すると消えます）

### 修正

- ターミナルの入力候補で「提案がありません」が頻発する問題を修正しました。zshのオートサジェスト（薄い文字）が入力の一部として誤認識され、候補の絞り込みが空振りしていたのが原因です。薄い文字の検出を強化し、候補を要求するときに正しく除外するようにしました
- モバイルアプリからClaude Codeのモデル/思考レベルを変更したとき、Claude側の確認ダイアログ（Switch model?）で切り替えが止まり、以降メッセージを受け付けなくなることがある問題を修正しました。モバイルからの切り替え時は確認が自動で承認されます（PC上で自分で操作した場合の確認には影響しません）
- モバイルアプリのメッセージ入力欄で、日本語入力の変換が途中で確定・破棄されたり濁点/半濁点が乱れることがある問題を修正しました（エージェントの応答受信中に特に発生していました）。あわせて応答ストリーミング中の画面更新を間引き、表示も軽くなりました
- Claude Codeがサブエージェント（Taskツール）を使ったとき、サブエージェントの完了だけで「作業完了」の通知（モバイルのプッシュ通知・PCの通知音/OS通知・Workspacesビューの完了ドット）が誤って発火する問題を修正しました。本体のターンが終わったときにだけ通知されます
- サイドバーのContainersビュー（Docker）のワークスペース絞り込みが、コンテナだけでなくイメージ・ボリューム・ネットワークにも効くようになりました（従来は他スペースのcomposeイメージや全ボリュームが表示されていました）。alpine等の共有ベースイメージやcompose以外のリソースは従来どおり常に表示されます。Container Tools拡張はパッチ版2.4.107に更新され、次回起動時に自動で置き換わります
- デスクトップでCodexを起動した時点で、モバイルアプリのホーム一覧に表示されるようになりました（従来は最初のメッセージへの応答が始まるまで表示されませんでした）
- HTMLファイルのプレビューで、背景色を指定していないHTMLの背景が透明になり、背後の画面が透けて内容が読めなくなることがある問題を修正しました。ブラウザと同じ白背景が敷かれます（HTML側で背景色が指定されている場合はそちらが優先されます）
- Codexターミナルタブ名の設定（`paradis.codex.terminalTitle.enabled`）の説明を実際の挙動に合わせて修正しました。タブ名は「最初のプロンプト」ではなく、Codexが会話内容から自動生成するスレッドタイトルで、生成が完了するまではスレッドID（UUID）が表示されます

## paracode-40（2026-07-12）

### 新機能

- Codex の統合ターミナルのタブ名が、Codex 名と最初のプロンプトの内容から自動で設定されるようになりました（例: 「codex: バグ修正」）。設定の「Para Code」セクション（`paradis.codex.terminalTitle.enabled`）でオフにできます（既定オン、設定変更後に起動した Codex セッションから反映）
- 統合ターミナルで `Shift+Enter` を押すと `Alt+Enter` と同じ扱いになり、Claude Code や Codex などで送信ではなく改行として入力できるようになりました。設定の「Para Code」セクション（`paradis.terminal.shiftEnterNewline`）でオンオフできます（既定オン）
- モバイルアプリのホーム画面で「レビュー」バッジをタップすると「確認済みにする」ポップオーバー（iOS 26以降はLiquid Glass表示）が開き、作業が終わったエージェントのステータスをその場でアイドルに戻せるようになりました。PC側でターミナルを開いて確認したときと同じ扱いになり、関連する通知も既読になります
- モバイルアプリのワークスペースドロワーの見出し右の「＋」から、新しいスペース（worktree）を作成できるようになりました。PC版の作成ダイアログと同じく、スペース名・ブランチ名・エージェントへの指示・エージェントの種類・リポジトリ・ベースブランチを指定でき、ブランチ名の自動生成（指示文から）やエージェントの自動起動もPC側と同様に行われます

### 改善

- サイドバーのContainersビュー（Docker）に、開いていないスペースのdocker composeコンテナが表示され続ける問題を改善しました。現在開いているワークスペースに関係するcomposeプロジェクトのコンテナだけが表示され、スペースを切り替えると即座に一覧が追従します（composeを使わず起動したコンテナは従来どおり常に表示されます）。設定 `containers.containers.scopeToWorkspace` でオフにできます（既定オン）。この対応のためContainer Tools拡張はPara Code同梱のパッチ版（2.4.106）に置き換わり、既にインストール済みの場合も次回起動時に自動で置き換わります（WindowsでWSL内のcomposeプロジェクトを `\\wsl$` パスで開いている構成にも対応）
- モバイルアプリのエージェント詳細画面の自動スクロールを見直しました。エージェントの動作中（応答の生成中やツール実行中）も画面が最新の内容へ追従し続けるようになり、逆に履歴を上へ遡って読んでいる間は新着が届いても勝手に最下部へ飛ばされなくなりました。遡り読み中は右下に「最新へ戻る」ボタン（新着件数付き）が表示され、タップで追従を再開できます
- モバイルアプリのホーム画面の長押しメニューと、そこから開く名前変更・削除確認ダイアログが、iOS 26以降でネイティブのLiquid Glass素材で表示されるようになりました（それ未満のiOSやAndroidでは従来同様のぼかし表示になります）
- モバイルアプリのソース管理タブで、リポジトリ名とブランチ名を2行（1行目リポジトリ名・2行目ブランチ名）に分け、長い名前同士が1行内で潰し合わないようにしました

### 修正

- モバイルアプリのソース管理タブの「最近のコミット」の経過時間表示が、取得した時点の値のまま古くなっていく問題を修正しました。表示のたびに再計算され、画面を開いたままでも1分ごとに追従します。あわせて表示を英語（"3 minutes ago"）から日本語（「3分前」）に統一し、rebaseしたコミットが実際より古い時刻で表示される問題も解消しました（コミット日時基準に変更）。通知一覧・Ccusageの経過時間表示も同様に追従するようになり、7日以上前の項目は日付表示（例: 6/28）に切り替わります（Ccusageの1分未満の表記は「たった今」から「今」に統一）
- モバイルアプリを開いた瞬間に、バックグラウンド中に溜まっていた過去の通知がまとめてOS通知（バナー）として表示されることがある問題を修正しました。発生から1分を超えた通知はバナーには出さず、アプリ内の通知一覧にだけ残ります
- エージェントの状態検知を安定化しました。ターミナルの再読み込みが終了と誤認されて状態がリセットされる問題や、閉じたターミナルの状態が残り続けることがある問題を修正し、Agent Teams使用時にチーム内のエージェント間メッセージがモバイルでユーザーの発言のように表示される問題も送信元付きの専用カード表示に改めました。あわせて、状態検知用フック設定の書き込みが他の設定変更と競合して片方が失われることがある問題も修正しました

## paracode-39（2026-07-12）

### 新機能

- ターミナルで Cmd+V を押したとき、クリップボードがスクリーンショット等の画像だけの場合に、Claude Code / Codex などのAIエージェントへ画像を添付できるようになりました（従来は Ctrl+V のみ）。テキストのペーストはこれまで通り動作します

- ワークスペース一覧のワークツリー行（local以外）を右クリックすると「Rename...」で表示名を変更できるようになりました

### 改善

- ワークスペース一覧の見た目を整理しました。リポジトリごとの色がワークツリー行まで途切れず表示されるようになり、リポジトリの見出し行にあった本アイコン（local行と重複していました）を削除し、代わりにリポジトリ間へ薄い区切り線を追加しました

## paracode-38（2026-07-12）

### 新機能

- Workspaces ビューのリポジトリ右クリックメニューに「Setup/Teardown Scripts...」を追加しました。リポジトリごとに setup / teardown スクリプトを設定すると、スペース（worktree）の作成直後・削除前に自動実行されます（設定はリポジトリの .paracode.json に保存され、コミットすればチーム全体で共有されます）
- ワークスペース一覧の各行に、作業ツリーの未コミット差分（+追加行数 / -削除行数。未追跡の新規ファイルは含まれません）が表示されるようになりました
- ワークスペース一覧で、リポジトリ本体（main checkout）もワークツリーと並ぶ1行（「local」）として表示されるようになりました。リポジトリの見出し行は展開/折りたたみと件数表示、ホバー時の「新規worktree作成」ボタンだけを持つグルーピング専用の行になりました
- ワークツリー行の右クリックメニューに「Copy Branch Name」「Move Up」「Move Down」を追加しました。並び順は手動で変更でき、次回起動後も保持されます
- モバイルアプリの設定に「PC作業中は通知しない」トグルを追加しました。ONにすると、PCの画面を見ている間に発生した完了・質問通知はモバイルへ一切配信されなくなります
- PC側でエージェントのペインを確認済みにすると（そのターミナルを開く、フォーカスして見るなど）、対応するモバイルの通知一覧からもその通知が自動的に消えるようになりました

### 改善

- モバイルアプリのブラウザを下部タブから独立させ、エージェント詳細画面のヘッダーにあるブラウザボタンから開くように変更しました。そのエージェントと共有中のブラウザページがあればボタンに緑ドットが表示され、開いたときにそのページが自動選択されます。複数ページがある場合は上部のタブチップで切り替えられます
- モバイルアプリのドロワーでワークスペースを選ぶと、ホームのエージェント一覧がそのワークスペース（とワークツリー）だけに絞り込まれるようになりました。ワークスペース一覧の上部に追加した「すべて表示」を選べば、これまで通り全ワークスペース横断の一覧に戻せます

### 修正

- 新しいスペース（worktree）作成直後のセットアップ中に別のスペースへ切り替えていると、セットアップ完了後に開いたターミナル（既定ターミナル・自動実行プリセット・エージェント起動）が、作成対象ではなく切り替え先の（今表示している）スペースに表示されてしまう問題を修正しました
- スペースを切り替えて使っているうちに、別のスペースで動いていたはずのターミナルが今開いているスペースの方に突然表示されることがある問題を修正しました
- エディタ領域にタブとして開いたターミナルが、スペースを切り替えて戻ってきたときに復元されず、シェルプロセスだけが見えないまま動き続けることがある問題を修正しました
- ターミナルを開いた直後にウィンドウをリロードすると、別のスペースに属していたターミナルが今開いているスペースに表示されてしまうことがある問題を修正しました
- スペース（worktree）の削除時、親リポジトリへの切り替えに失敗した場合は削除を中止するようになりました。これまでは開いているフォルダがディスクから削除され、ウィンドウが空のまま操作不能になることがありました
- 通知設定ダイアログで、音量やトグルなど無関係な設定を変更しただけでAivisの辞書一覧・使用量セクションまで再読み込みされ、スクロール位置がずれたり画面がちらついたりする問題を修正しました
- Aivis読み上げのフォーマット欄でプレースホルダーのボタンをクリックしても挿入されなかったり、カーソル位置とは違う場所に入ってしまったりする問題を修正しました
- モバイルアプリの通知一覧で、項目をタップしてターミナルへ遷移してもその通知が一覧に残り続ける問題を修正しました。タップした通知は一覧から消え、複数端末をペアリングしている場合は他の端末の一覧からも同期して消えます
