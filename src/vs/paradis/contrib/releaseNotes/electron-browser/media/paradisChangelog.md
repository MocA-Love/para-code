# Para Code 更新履歴

Para Code が本家 VS Code に加えた変更の一覧です。新しいバージョンが上に来ます。

## paracode-7（2026-07-04）

- ワークスペースを何度も切り替えると、ソース管理ビューに別のスペースのリポジトリが表示されたままになることがある問題を修正しました（絞り込みを常時強制するようになったため、スコープ外リポジトリを一時的に手動表示することはできなくなりました。全リポジトリを見たい場合は設定 `paradis.workspaceSwitch.scopeScmRepositories` をオフにしてください）
- コミットメッセージの日本語生成が実際には効いていなかった問題を修正しました（同梱 Copilot 拡張が既定値レイヤーを読まないため、初回起動時に user settings.json へ日本語指示を書き込む方式に変更。既に自分で設定している場合は上書きしません）
- ワークスペースを切り替えるとターミナルペインの識別子が失われ、ブラウザページの共有（para-browser MCP）が二度とできなくなる問題を修正しました
- ブラウザページが未共有のときの para-browser MCP の案内文に、アプリ再起動（自動アップデート適用を含む）で共有がリセットされるため再共有が必要である旨を追記しました

## paracode-6（2026-07-04）

- Source Control の「コミットメッセージを生成」が日本語でメッセージを作るようになりました（設定 `github.copilot.chat.commitMessageGeneration.instructions` を独自の指示で上書きできます）
- 既定テーマ（Houston / Dark 2026 / Light 2026）のボタン・バッジ・フォーカス枠などのアクセント色を Para Code のブランドカラー（#09AFD9）に統一し、ステータスバーも通常時・フォルダ未オープン時ともブランドカラーになりました（settings.json の `workbench.colorCustomizations` でいつでも上書きできます）
- 初回セットアップ直後に拡張機能の Welcome ページ（Welcome to Dependi、Deno for VSCode、Get Started など）が大量に開かないようにしました
- 既定でインストールされる拡張機能から Swift を外しました（インストール済みの環境からは削除されません。不要な場合は拡張機能ビューからアンインストールしてください）
- 右サイドバーの既定の並び順を Explorer → Source Control → Chat に変更しました（新規インストール時の既定。既存環境はアイコンのドラッグで並び替えできます）
- 内蔵ブラウザに React Developer Tools を標準搭載しました。ブラウザの「Developer Tools」を開くと React コンポーネントツリー（⚛️ Components / Profiler タブ）を検査できます
- ターミナルのエージェント CLI から使う Para Browser MCP（para-browser）が、パッケージ版で接続に失敗する問題を修正しました（MCP 中継スクリプトが配布物に含まれていませんでした）

## paracode-5（2026-07-04）

- ウィンドウ透過を既定で有効にしました（既定の不透明度は 0.8）。設定 `paradis.window.transparency.enabled` でオフにできます
- エディタを何も開いていないとき（ショートカット一覧が出るトップページ）も、他の領域と同じように背景が透けるようにしました

## paracode-4（2026-07-04）

- Workspaces ビューのスペース名の下に、ディレクトリパスの代わりにチェックアウト中のブランチ名を表示するようにしました。行の高さも少し広げて見やすくしました
- パッケージ版で「更新履歴」を開くと「ファイルが見つかりません」になる問題を修正しました（更新後の自動表示もこれで動くようになります）

## paracode-3（2026-07-04）

- ソース管理ビューの表示を、現在開いているスペース（または worktree）に関係するリポジトリだけに自動的に絞るようにしました。切り替え前のスペースや worktree の親リポジトリが並んだままにならなくなります（設定 `paradis.workspaceSwitch.scopeScmRepositories` でオフにできます）
- Toggle Terminal（⌃⌘T）が「アクティブなエディタグループ」を対象になりました。Split した直後の空のグループで押すと、そのグループに新しいターミナルが開きます
- 設定の「Para Code」セクションの説明文をすべて日本語にしました
- 歯車メニューに「更新履歴」を追加しました（このページです）
- 自動更新の適用後に再起動すると、更新履歴が自動的に開くようになりました（設定 `paradis.releaseNotes.showOnUpdate` でオフにできます）

## paracode-2（2026-07-03）

### 改善

- 初回起動のオンボーディング画面を Para Code の名前とロゴに変更しました
- 初回起動時に左サイドバーは Workspaces、右サイドバーは Explorer が開くようになりました
- Workspaces ビューの操作性を改善しました
  - スペース名のクリックで常にメインのワークスペースへ切り替わります（worktree 一覧の開閉は左の開閉ボタンのみ）
  - スペースの色バーを行の左端に移動しました
  - 色の選択時にカラースウォッチ（色見本）が表示されるようになりました
- ターミナルを含むエディタグループが自動的にロック（鍵マーク）されないようにしました

### 修正

- パッケージ版で初回の拡張機能自動インストールが失敗する問題を修正しました
- ワークスペースの切り替えでエディタ領域のターミナルが壊れる問題を修正しました（切り替えを跨いでターミナルが生き続けます）
- リリース工程の不具合（macOS 配布物のアップロード失敗など）を修正しました

## paracode-1（2026-07-03）

Para Code の初回リリースです。VS Code 1.128 をベースに、以下の独自機能を追加しています。

### 新機能

- **ワークスペースの即時切り替え**: 左サイドバーの Workspaces ビューに複数のリポジトリや git worktree を登録し、ワンクリック（または ⌃⌘1〜9）で切り替えられます。開いていたエディタ・ターミナル・コミットメッセージの下書きは切り替え先ごとに保持されます。スペースごとの色分け・名前変更・エージェント稼働状態の表示にも対応
- **ターミナルの 2D グリッド分割**: ターミナルを上下左右の 4 方向に分割し、ドラッグ & ドロップで自由に配置できます
- **内蔵ブラウザ**: エディタ内でブラウザを開けます（Toggle Browser）。ブックマークバー、AI エージェントとの連携（ページやターミナルをエージェントに割り当て）に対応
- **レンダリングファイルビューア**: HTML / Markdown / Excel（xlsx）をレンダリングした状態で表示できます。Excel はセル単位の git 差分表示に対応
- **通知**: エージェントの作業完了などを通知音と Aivis の音声読み上げで知らせます
- **リソースモニター**: タイトルバーに CPU / メモリ使用率を表示します
- **ターミナル履歴補完**: シェルを横断した永続コマンド履歴から補完できます（↓キーで候補を表示）
- **ウィンドウ透明度**: 背景だけを半透明にしてデスクトップを透かせます（設定の「Para Code」セクション）
- **セカンダリアクティビティバー**: 右サイドバーにもアクティビティバーを追加し、既定で Source Control を配置

### 変更

- アプリアイコンとアクセントカラーを Para Code 独自のデザインにしました
- 初回起動時に既定の拡張機能セットを自動インストールします
- 独自の配布・自動アップデート基盤に接続します（macOS / Windows / Linux）
